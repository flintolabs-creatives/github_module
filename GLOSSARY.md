# GitHub Glossary üìñ

Common terms you'll hear when working with Git and GitHub.

## üî§ The Basics

### Repository (Repo)
A project folder that Git tracks. Contains all your code, files, and the history of changes.
**Think of it as:** A project folder with superpowers - it remembers every change ever made!

### Clone
Making a copy of a repository from GitHub to your computer.
**Think of it as:** Downloading a project to work on locally.

### Commit
Saving a snapshot of your changes with a message describing what you did.
**Think of it as:** A save point in a video game - you can always go back to it!

### Push
Uploading your commits from your computer to GitHub.
**Think of it as:** Backing up your work to the cloud.

### Pull
Downloading the latest changes from GitHub to your computer.
**Think of it as:** Getting updates that others made.

### Branch
A separate version of your code where you can make changes without affecting the main version.
**Think of it as:** A parallel universe where you can experiment safely.

### Merge
Combining changes from one branch into another.
**Think of it as:** Bringing your experimental work back into the main project.

---

## üåø Working with Branches

### Main Branch (or Master)
The primary branch of your repository. Usually contains the "official" working version.
**Think of it as:** The main storyline in a game.

### Feature Branch
A branch created to work on a specific feature or change.
**Think of it as:** A side quest where you build something new.

### Checkout
Switching from one branch to another.
**Think of it as:** Switching between different saved games.

### HEAD
A pointer to your current location (which branch and commit you're on).
**Think of it as:** Your "You Are Here" marker.

---

## üíæ Saving Your Work

### Staging (or Index)
Preparing files to be committed. You choose which changes to include.
**Think of it as:** Deciding which items to put in your shopping cart before checkout.

### Working Directory
The folder on your computer where you edit files.
**Think of it as:** Your desk where you do the actual work.

### Commit Message
A short description of what changes you made in a commit.
**Think of it as:** A note to your future self (and teammates) explaining what you did.

---

## üîÑ Syncing Changes

### Fetch
Checking if there are new changes on GitHub without downloading them yet.
**Think of it as:** Checking your mail without opening the letters.

### Remote
The version of your repository stored on GitHub (online).
**Think of it as:** The cloud backup of your project.

### Origin
The default name for the remote repository you cloned from.
**Think of it as:** The nickname for your project's home on GitHub.

### Upstream
The original repository that you forked from (advanced topic).
**Think of it as:** The source of truth when working on someone else's project.

---

## üîÄ Collaboration

### Fork
Making your own copy of someone else's repository.
**Think of it as:** Photocopying a recipe to modify for yourself.

### Pull Request (PR)
Asking the project owner to review and merge your changes.
**Think of it as:** Submitting your homework for review.

### Review
When someone looks at your code and gives feedback.
**Think of it as:** Peer editing for code.

### Conflict
When two people change the same part of a file differently.
**Think of it as:** Two authors trying to edit the same sentence.

### Merge Conflict
A conflict that you need to resolve manually before merging.
**Think of it as:** Deciding which edit to keep when two people changed the same thing.

---

## üìä Tracking Changes

### Diff
The difference between two versions of a file.
**Think of it as:** Track changes in Microsoft Word, but for code.

### History
A log of all commits made to a repository.
**Think of it as:** A complete timeline of everything that happened to the project.

### Blame
Seeing who made each change to a file (not as mean as it sounds!).
**Think of it as:** Finding out who wrote which part of the code.

---

## üõ†Ô∏è Special Files

### .gitignore
A file that tells Git which files to ignore (not track).
**Think of it as:** A "do not disturb" list for certain files.

### README.md
A file that explains what your project is about.
**Think of it as:** The cover page of your project.

### .md (Markdown)
A simple way to format text files (like this glossary!).
**Think of it as:** A simpler version of Word formatting.

---

## ‚öôÔ∏è Advanced Concepts

### SHA / Hash
A unique ID for each commit (looks like: `a3f5b2c`).
**Think of it as:** A barcode for each save point.

### Stash
Temporarily saving changes without committing them.
**Think of it as:** Putting your work in a drawer to clean up your desk.

### Rebase
A way to reorganize commits (advanced - ask instructor first!).
**Think of it as:** Reorganizing your timeline (use with caution!).

### Tag
A label for a specific commit (often used for versions like v1.0).
**Think of it as:** A bookmark for an important moment.

---

## üö´ What NOT to Do

### Force Push
Forcing your changes to overwrite others (dangerous!).
**What it is:** The nuclear option - use ONLY if instructor says to.

### Rewrite History
Changing past commits (very advanced and risky).
**What it is:** Time travel - usually a bad idea when working with others.

---

## üí¨ Common Phrases

### "Push your changes"
Upload your commits to GitHub.

### "Pull latest"
Download the newest version from GitHub.

### "Create a branch"
Make a separate workspace for your changes.

### "Commit your work"
Save a snapshot of your changes.

### "Merge conflicts"
Two people edited the same thing - need to decide which to keep.

### "The repo"
Short for repository - the project you're working on.

### "Make a PR"
Create a pull request to suggest your changes.
