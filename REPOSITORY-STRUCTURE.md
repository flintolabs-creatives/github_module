## ğŸ“ File Purposes

### Core Documentation
- **README.md**: First thing students see - explains what the repo is for
- **CONTRIBUTING.md**: Guidelines for making contributions (branch naming, commit messages, etc.)
- **QUICK-REFERENCE.md**: Handy cheat sheet they can refer to while working
- **GLOSSARY.md**: Definitions of all Git/GitHub terminology
- **CHALLENGES.md**: Structured exercises to practice GitHub skills

### Configuration
- **.gitignore**: Teaches students what types of files shouldn't be committed (API keys, system files, etc.)

### Practice Files
- **students.txt**: Simple first task - add your name
- **about-me/**: Practice creating files and writing markdown
- **my-first-code/**: Practice editing code files with clear TODOs
- **projects/**: Showcase their AI work from Flintolabs

## ğŸ¯ Learning Progression

### Week 1: Basics
Students will:
1. Clone the repository
2. Add their name to students.txt
3. Create their profile in about-me/
4. Make their first commits

### Week 2: Practice
Students will:
1. Edit Python, HTML, and JavaScript files
2. Complete TODO items
3. Practice multiple commits
4. Get comfortable with the workflow

### Week 3: Projects
Students will:
1. Document their AI projects
2. Practice good README writing
3. Share their work with the class

### Ongoing: Challenges
Students can work through CHALLENGES.md at their own pace to build confidence.
